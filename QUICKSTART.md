# Локальный запуск — пошагово

Это инструкция для первого запуска бота у себя. Тест ограничивается Telegram-частью (`/start`, дисклеймер, `/register`, `/status`, `/start_monitor`, `/pause`, `/stop`). Реальный мониторинг и бронирование Prenot@Mi включаются только после калибровки селекторов под живой сайт.

## Требования

- Docker и Docker Compose v2 (`docker compose version` отвечает > 2.x).
- Аккаунт в Telegram, доступ к [@BotFather](https://t.me/BotFather).
- Никаких других зависимостей — Python, Postgres, Redis и Playwright поднимаются в Docker.

## Шаг 1 — Получить токен бота

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram.
2. `/newbot` → имя (например, `My Italy Visa Bot`) → username (должен заканчиваться на `bot`, например, `my_italy_visa_bot`).
3. BotFather пришлёт строку вида `8871599353:AAE...`. Это токен.
4. **Никому не пересылайте токен.** Если случайно отправили в чат или в скриншоте — `/mybots` → ваш бот → `API Token` → `Revoke current token`.

## Шаг 2 — Клон и `.env`

```bash
git clone https://github.com/alloben812/Bot_embassy.git
cd Bot_embassy
git checkout main

cp .env.example .env
```

Откройте `.env` в редакторе и заполните:

- `BOT_TOKEN` — токен от BotFather.
- `BOT_ENCRYPTION_KEY` — сгенерируйте Fernet-ключ:

  ```bash
  python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
  ```

  Скопируйте вывод в `BOT_ENCRYPTION_KEY`. **Этот ключ должен быть постоянным**: если потеряете — все сохранённые креды в БД станут нечитаемыми.

Остальные переменные оставьте по умолчанию.

## Шаг 3 — Поднять Postgres и Redis

```bash
docker compose up -d postgres redis
docker compose ps
```

Должны быть `running (healthy)` для обоих.

## Шаг 4 — Накатить миграции

```bash
docker compose run --rm bot alembic upgrade head
```

В выводе должно появиться `Running upgrade  -> 0001_initial`. После этого таблицы созданы.

## Шаг 5 — Запустить бота (без воркера на этом этапе)

```bash
docker compose up bot
```

В логах должно быть `Starting Telegram bot` и далее тишина (long-polling без апдейтов). Ошибок типа `Unauthorized` быть не должно — если есть, проверьте `BOT_TOKEN`.

## Шаг 6 — Проверить в Telegram

Найдите бота по username, откройте чат и пройдите сценарий:

1. **`/start`**
   - Должно прийти приветствие.
   - Следующее сообщение — текст дисклеймера с двумя кнопками: «Подтверждаю резидентство в Сербии» и «Принимаю риски использования бота».
2. **Тапните по очереди обе кнопки.** Их текст должен поменяться на «Подтверждено: …», и появится третья кнопка «Согласиться и начать регистрацию».
3. **Тап «Согласиться»** → сообщение `Согласие записано. Начнём регистрацию: /register`.
4. **`/register`** → запросит email Prenot@Mi. Введите любой валидный email (можно тестовый, реальный счёт не нужен на этом шаге).
5. Дальше FSM проведёт через:
   - email → пароль (ваше сообщение с паролем будет удалено)
   - ФИО (латиницей) → паспорт → дата рождения `ДД.ММ.ГГГГ` → гражданство `RU`
   - выбор статуса в Сербии (4 кнопки) → номер документа
   - дата с / дата по в формате `ДД.ММ.ГГГГ`
   - выбор дней недели (мульти-выбор, потом «Готово»)
   - сводка → «Сохранить»
6. **`/status`** → покажет задачу #1 со статусом `pending`.
7. **`/start_monitor`** → задача переходит в `active`, бот ответит `Мониторинг запущен`. (Воркер пока не запущен — задачу никто реально не выполнит, это нормально.)
8. **`/pause`** → `paused`, **`/stop`** → удаляет апликанта и задачу.

## Шаг 7 — Запустить воркер (опционально, для теста цикла)

Без калибровки селекторов воркер не сможет реально найти слот, но цикл проверки/логирования отработает (увидите попытки в таблице `slot_attempts`):

```bash
docker compose up worker
```

При первой проверке Playwright откроет Chromium, дойдёт до страницы логина, дальше упрётся в реальную форму Prenot@Mi. Результат запишется как `auth_error` или `site_changed` в `slot_attempts` — это ожидаемо до калибровки.

## Шаг 8 — Остановиться

```bash
docker compose down
```

Данные в Postgres сохраняются в volume `postgres_data`, при следующем `up` всё на месте.

## Проблемы

- **`Unauthorized` в логах бота** — неверный токен или бот удалён через BotFather. Перевыпустить токен, обновить `.env`, перезапустить.
- **`Connection refused` на Postgres** — `docker compose ps` показывает, что Postgres не `healthy`. Дождитесь полного старта (`logs postgres`) и повторите миграцию.
- **Бот не отвечает на `/start`** — проверьте, что только один контейнер бота запущен (`docker compose ps`), что в Telegram открыт правильный бот (URL `t.me/<username>`), и что в логах нет ошибок.
- **Сообщение с паролем не удалилось** — у бота должно быть право удалять сообщения в чате. В личных чатах с ботом это работает по умолчанию.

## Что покажет live test

- Корректность UX-флоу (приветствие, дисклеймер, FSM регистрации).
- Удаление сообщения с паролем.
- Шифрованное сохранение кредов и профиля в Postgres (можно проверить: `docker compose exec postgres psql -U embassy -d embassy -c 'SELECT id, serbia_status_type, length(credentials_encrypted) FROM applicants;'` — увидите бинарь, не plaintext).
- Работу команд `/status`, `/pause`, `/stop`, переключение статуса задачи.

Что НЕ покажет (до отдельной калибровки):
- Реальный логин в Prenot@Mi.
- Поиск слотов в календаре.
- OTP-relay (без логина не дойдём до OTP).
- Бронирование.
